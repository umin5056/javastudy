
<Getter와 Setter>

1. Getter와 Setter의 메소드는 이름 형식이 정해져있으면 개발자가 임의로 수정해서는 안된다.
2. boolean 타입의 필드는 Getter의 이름이 is+필드이다.
    (단, isVip같이 앞에 is를 사용했을 경우 앞에 추가로 is가 생성되지 않는다.)
3. 이클립스에서 자동으로 메소드를 생성해준다.

<static>

1. static 메소드는 클래스에서 호출하기 때문에 클래스메소드라고도 한다.
   ex. Integer.parseInt( );
2. static은 해당 클래스의 객체에서 공유되는 자원이다.
3. static 필드는 private을 쓰지 않고, public을 사용한다.
    (static은 하나의 자원을 공유. private 자원의 공유를 막고 간접 접근만 허용하기 때문)
4. static으로 지정된 변수와 메소드는 객체로 호출하지 않고 클래스로 직접 호출한다.
    (객체를 이용해 호출이 가능하지만, 툴에서 경고를 보낸다.)
5. static 요소는 static만 사용할 수 있다.
    ex.static 메소드에는 static 필드만 사용할 수 있다.
    (static은 객체 생성 이전에 클래스에서 만들어지는데,
     일반 필드는 객체를 생성해야 만들어지므로, 생성되지 않는 값을 사용하기 때문)
